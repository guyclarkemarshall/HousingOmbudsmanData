# Governance

This project follows a **benevolent stewardship** model to ensure professional engineering standards and open data representation for the UK social housing sector.

## Steering Group
The strategic direction of the project is guided by the project owner:
- **Guy Clarke Marshall** (@guyclarkemarshall) - Primary steward.

Interested housing associations, policy units, or local authorities who contribute data insights or code can request representation on the steering group by opening a GitHub Discussion.

## Maintainers
Technical direction, code review, and merge authority reside with:
- **Guy Clarke Marshall** (@guyclarkemarshall)

## How Decisions Are Made

| Decision Type | Who Decides | How |
|---------------|-------------|-----|
| Bug fix / scraper tweak | Maintainers | Pull Request review and approval |
| Database schema change | Maintainers + Steering Group | Issue discussion followed by a PR with migration logs |
| Change to classifications | Maintainers + Steering Group | Technical discussion in issues or PRs |
| Dependency addition | Maintainers | PR review |

## How to Become a Maintainer
We follow a progressive contributor ladder (see [CONTRIBUTING.md](CONTRIBUTING.md)). Consistent, high-quality contributions (issue triage, documentation improvement, bug fixes, or dataset testing) will lead to an invitation to join as a Maintainer.

## Code of Conduct
All participants are expected to adhere to our [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
